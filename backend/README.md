# AndroidPaint Cloudflare Worker 后端

基于 Cloudflare Workers 的后端 API，为 AndroidPaint 插画应用提供云同步功能。

## 功能特性

- **用户认证**：邮箱注册/登录、手机号登录、微信授权登录
- **项目管理**：创建、读取、更新、删除项目元数据
- **云端同步**：版本控制、冲突检测、增量同步
- **文件存储**：R2 存储 ORA 项目文件和缩略图

## 快速开始

### 前置条件

1. 安装 Node.js 18+
2. 注册 Cloudflare 账号并完成实名认证
3. 安装 Wrangler CLI：`npm install -g wrangler`

### 部署步骤

1. **克隆项目并安装依赖**

```bash
cd backend
npm install
```

2. **创建 KV 命名空间**

```bash
wrangler kv:namespace create "ANDROID_PAINT_KV"
```

将返回的 `id` 填入 `wrangler.toml` 的 `[[kv_namespaces]]` 配置中。

3. **创建 R2 存储桶**

```bash
wrangler r2 bucket create androidpaint-projects
```

4. **配置环境变量**

编辑 `wrangler.toml` 中的 `[vars]` 部分：

```toml
[vars]
JWT_SECRET = "your-super-secret-jwt-key-change-in-production"
```

> ⚠️ **重要**：生产环境必须使用强密钥！

5. **部署到开发环境**

```bash
npm run dev
```

6. **部署到生产环境**

```bash
npm run deploy
```

## API 文档

### 认证接口

#### 邮箱注册
```
POST /api/auth/register
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "Password123"
}

Response:
{
  "success": true,
  "userId": "uuid",
  "accessToken": "jwt-token",
  "refreshToken": "refresh-token"
}
```

#### 邮箱登录
```
POST /api/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "Password123"
}
```

#### 发送手机验证码
```
POST /api/auth/phone/send
Content-Type: application/json

{
  "phone": "13800138000"
}

Response (开发阶段):
{
  "success": true,
  "message": "验证码已发送",
  "devCode": "123456"  // 开发阶段返回验证码方便测试
}
```

> **注意**：实际短信发送需要接入阿里云短信服务，当前为开发阶段，仅日志输出验证码。

#### 手机号登录
```
POST /api/auth/phone/login
Content-Type: application/json

{
  "phone": "13800138000",
  "code": "123456"
}
```

#### 微信登录
```
POST /api/auth/wechat
Content-Type: application/json

{
  "code": "wechat-authorization-code"
}
```

> **注意**：需要配置微信开放平台的 AppID 和 AppSecret。

#### 刷新 Token
```
POST /api/auth/refresh
Content-Type: application/json

{
  "refreshToken": "refresh-token"
}
```

### 项目接口

所有项目接口需要在请求头中携带 Access Token：
```
Authorization: Bearer <access-token>
```

#### 获取项目列表
```
GET /api/projects
```

#### 创建项目
```
POST /api/projects
Content-Type: application/json

{
  "name": "我的作品",
  "width": 1080,
  "height": 1920
}
```

#### 获取项目详情
```
GET /api/projects/:id
```

#### 更新项目
```
PUT /api/projects/:id
Content-Type: application/json

{
  "name": "新名称",
  "layerCount": 5
}
```

#### 删除项目
```
DELETE /api/projects/:id
```

#### 上传项目文件
```
PUT /api/projects/:id/upload
Content-Type: application/octet-stream
X-Device-Id: device-identifier

<binary-ora-file>
```

#### 下载项目文件
```
GET /api/projects/:id/download
```

### 同步接口

#### 获取同步状态
```
GET /api/sync/status?projectId=xxx
```

#### 推送同步
```
POST /api/sync/push
Content-Type: application/json

{
  "projectId": "xxx",
  "version": 3,
  "deviceId": "device-identifier"
}

Response (冲突):
{
  "success": false,
  "conflict": true,
  "localVersion": 2,
  "remoteVersion": 4,
  "remoteTimestamp": 1699999999,
  "message": "检测到版本冲突"
}
```

#### 拉取同步
```
POST /api/sync/pull
Content-Type: application/json

{
  "projectId": "xxx",
  "version": 2
}

Response:
{
  "success": true,
  "hasUpdate": true,
  "version": 4,
  "hasFile": true,
  "metadata": { ... }
}
```

#### 解决冲突
```
POST /api/sync/resolve
Content-Type: application/json

{
  "projectId": "xxx",
  "resolution": "local"  // 或 "remote"
}
```

## 数据结构

### KV 存储

```
# 用户数据
user:{userId} = { id, email, phone, wechatOpenId, salt, passwordHash, createdAt }

# 邮箱索引
email:{email} = userId

# 手机号索引
phone:{phone} = userId

# 微信索引
wechat:{openid} = userId

# 验证码
sms:{phone} = { code, expiresAt }  # 5分钟过期

# 项目元数据
project:{userId}:{projectId} = { id, name, width, height, layerCount, fileSize, version, createdAt, modifiedAt }

# 项目版本
version:{userId}:{projectId}:{version} = { timestamp, deviceId }

# 用户项目列表
projects:{userId} = [projectId1, projectId2, ...]
```

### R2 存储

```
{userId}/{projectId}.ora       # ORA 项目文件
{userId}/{projectId}.thumb.png  # 缩略图
```

## 安全说明

1. **密码存储**：使用 PBKDF2 + Salt 哈希（100000 次迭代）
2. **JWT Token**：
   - Access Token：7 天有效期
   - Refresh Token：30 天有效期
3. **验证码**：5 分钟有效期，单次使用
